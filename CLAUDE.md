## Project Overview

NIP-101 is the **NOSTR relay filtering and synchronization layer** for the UPlanet/Astroport ecosystem. It provides:
- A **strfry writePolicy plugin** that filters incoming NOSTR events by kind, using per-kind bash scripts
- **Constellation backfill** (N² sync) between Astroport relay nodes via negentropy protocol
- Hardware-auto-tuned strfry configuration (x86_64 / aarch64 / RPi Zero)

## Dependencies

- Requires `~/.zen/Astroport.ONE` installed (provides `tools/my.sh`, `tools/keygen`, etc.)
- strfry binary installed at `~/.zen/strfry/strfry` (fetched from IPFS or compiled)
- `jq`, `ipfs`, `nostr_send_note.py`, `libsecp256k1-1`
- Python 3 with `websockets` (for `nostr_websocket_backfill.py`)

## Architecture

### Write Policy Plugin (`relay.writePolicy.plugin/`)

strfry pipes incoming events as JSON to the writePolicy plugin via stdin. The plugin reads each line, decides accept/reject, and writes the decision to stdout.

**Two entry points** (configured in `strfry.conf` via `writePolicy.plugin`):
- `all_but_blacklist.sh` — Default. Accepts all events except from blacklisted pubkeys. Classifies users as `nobody`/`player`/`uplanet` and applies per-kind filters.
- `process.sh` — Stricter. Only accepts events from pubkeys found in `~/.zen/game/nostr/*/HEX` (MULTIPASS holders).

**Event flow:**
```
strfry → stdin JSON → all_but_blacklist.sh
                        ├── is_key_blacklisted()? → reject
                        ├── classify_user() → nobody/player/uplanet
                        ├── filter/{kind}.sh → kind-specific logic (exit 0 = accept, exit 1 = reject)
                        └── stdout: {"id": "...", "action": "accept|reject|shadowReject"}
```

### Per-Kind Filter Scripts (`relay.writePolicy.plugin/filter/`)

Each `{kind}.sh` receives the full event JSON as `$1`. Must `exit 0` to accept, `exit 1` to reject. All source `common.sh` for shared functions.

### Common Functions (`filter/common.sh`)

Shared library sourced by all filter scripts. Key functions:
- `extract_event_data "$json"` — Sets `$event_id`, `$pubkey`, `$content`, `$created_at` via eval
- `get_key_email "$pubkey"` — Finds email for a pubkey from `~/.zen/game/nostr/*/HEX`
- `search_swarm_for_pubkey "$pubkey"` — Searches swarm directories
- `check_amis_of_amis "$pubkey"` — Checks `~/.zen/strfry/amisOfAmis.txt`
- `check_authorization "$pubkey" log_func` — Consolidated auth check (local → swarm → amisOfAmis), sets `$AUTHORIZED`, `$EMAIL`, `$SOURCE`
- `extract_tags "$json" tag1 tag2...` — Extracts multiple tag values in one jq call
- Crowdfunding helpers: `is_crowdfunding_bien`, `record_crowdfunding_contribution`, `record_assets_vote`

### Key File Locations (Runtime)

- `~/.zen/strfry/strfry.conf` — Generated by `setup.sh`
- `~/.zen/strfry/strfry-db/` — LMDB database
- `~/.zen/strfry/blacklist.txt` — Blocked pubkeys (fed by IA responder)
- `~/.zen/strfry/amisOfAmis.txt` — Extended trust circle (friends-of-friends)
- `~/.zen/game/nostr/*/HEX` — Authorized MULTIPASS pubkeys
- `~/.zen/tmp/strfry.log` — Main relay filter log
- `~/.zen/tmp/nostr_kind*.log` — Per-kind logs

### Constellation Backfill (`backfill_constellation.sh`)

Discovers peer relays from the Astroport IPFS swarm (`~/.zen/tmp/swarm/*/12345.json`), then uses `strfry sync` (negentropy protocol) and WebSocket REQ fallback (`nostr_websocket_backfill.py`) to synchronize events across the constellation. Runs automatically via Astroport's `_12345.sh` after 12:00.
